## SECTION 4 — SAFETY GATES (NON-NEGOTIABLE)

For **any story involving financial operations** (Alpaca API, order placement):

```typescript
// AlpacaGateway.ts — NEVER bypass this
if (process.env.ALPACA_PAPER !== 'true') {
  throw new Error('SAFETY GATE: ALPACA_PAPER must be true. Refusing to execute.');
}
```

- Unit tests MUST mock the Alpaca client. Never hit live API in tests.
- Log every order intent with timestamp before execution.
- Circuit breaker: if >3 errors in 60s, halt and post WARNING to memory DB.
- Write to context store: `mem.setContext('alpaca_paper_verified', 'true', 'global')`

For **any story touching VPS / SSH**:
- Never disable Fail2ban, UFW, or root login lockout.
- Never write credentials to disk unencrypted.
- Never write credentials to forge-memory.db.

For **any story with user-facing data**:
- Validate all inputs at the TypeScript boundary with zod.
- No `any` types in public interfaces.
